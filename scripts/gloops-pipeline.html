2. 🧬 Le Generateur de Personnage

Pipeline au chargement (7 etapes)

  1. Load FBX — Gloops_skeleton.fbx via Character.load()
  2. Capture T-pose — position/quaternion/scale de chaque bone + matrices monde inversees (_baseBoneInverses)
  3. Scan materials — ShadingManager.scanModel() remplace par MeshPhysicalMaterial
  4. Auto-connect textures — lit manifest.json → autoConnect et branche normalMap, roughnessMap, rgbMask, patterns
  5. Apply defaults — lit character.json → materialDefaults
  6. Load animations — lit manifest.json → categories, prune overlay, cree les actions
  7. Build UI — ANIM, MAT, PROPS, GENERATE tabs

Le ShadingManager (compositing CPU)

Chaque materiau du Gloop a un pipeline de compositing CPU sur un canvas 512x512 :

  1. RGB Mask (body_ID.png) — 3 zones (R, G, B) chacune avec Color A + Color B
  2. Blend Mask (body_Ramp.png) — gradient qui mixe entre A et B dans chaque zone
  3. Pattern (body_Pattern.N.png) — overlay multiplicatif avec hue/sat shift
  4. Normal / Roughness / AO / SSS / Displacement — branches directement sur le material three.js

Le resultat est bake dans une texture canvas, assignee comme map du MeshPhysicalMaterial.

3. 🎞️ Categories d'animation & pruning

Les animations sont mixees en overlay via l'AnimationMixer three.js. Chaque categorie a des regles de pruning qui enlevent les tracks de bones/morphs qui ne lui appartiennent pas.

CategorieDossierTypeBones gardes (prune)
EmotionANIM/Emotion/Base animTOUT (1ere categorie = base)
MoveANIM/Move/Full-bodyRAW, aucun filtrage
TeethsANIM/Teeths/OverlayTeeth + morphs teeths_low
HornsANIM/Horns/OverlayHorns, BS_Horns, Sk_Main_Horns
EyesANIM/Eyes/OverlaySk_Main_Eye_Lt, Sk_Main_Eye_Rt (exact)

Convention de nommage FBX

  • Emotion : 01_peace.fbx a 16_annihilated.fbx (numerotees par mood)
  • Move : walk.fbx, run.fbx, run_panic.fbx, walk_stupid.fbx, shake.fbx
  • Teeths : nom du style (Baby.fbx, Vampire.fbx, Sharp.fbx...)
  • Horns : nom du chapeau/cornes (Hat_CowBoy.fbx, kakarot.fbx, bald.fbx...)
  • Eyes : nom du style d'yeux (Aviator.fbx, Pirate.fbx, Normal.fbx...)

Prefixe exact: dans le pruning

Dans character.js, la config des patterns accepte un prefixe exact: pour forcer l'egalite stricte au lieu d'un substring match. Utile pour Eyes ou seulement les 2 joints parents doivent bouger, pas leurs enfants.
