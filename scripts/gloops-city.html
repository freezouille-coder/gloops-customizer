🏙️ City Blocks — Théorie de la ville procédurale

La ville est composée de blocs modélisés à l'avance. Chaque bloc intègre déjà la moitié d'une route tout autour de lui. Deux blocs collés forment une route à 2 voies ; quatre blocs qui se rencontrent à un coin forment un carrefour.

1. 🧩 Principe de base

Un bloc contient :

  • Le bâtiment (et son trottoir intérieur) au centre
  • Une demi-route de 5 m sur chacun des 4 côtés
  • Un quart de carrefour à chaque coin
Anatomie d'un bloc M (50×50 m). Bâtiment 40×40 m au centre, demi-route 5 m sur chaque côté, quart de carrefour à chaque coin.

2. 📏 Grille & unités

Cellule de base : 25 m. Tout bloc mesure N × M cellules.

Demi-route : 5 m comprise dans le bloc, sur chaque côté.

Route complète (2 voies) : 10 m = la somme des demi-routes de 2 blocs adjacents.

Carrefour : 10×10 m = somme des 4 quarts de carrefour de 4 blocs qui se rencontrent au coin.

Deux blocs adjacents → les demi-routes se combinent en une route complète 10 m, 2 voies.
Quatre blocs qui se rencontrent en coin → les 4 quarts de carrefour s'assemblent en 1 intersection complète.

4. 📝 Convention de nommage

block_<W>x<H>_<Name>[_w<weight>].<ext>

W       = largeur en cellules (1..4)
H       = profondeur en cellules (1..4)
Name    = label alphanumérique — utilise un vrai nom descriptif.
          Exemples : Stores, SuperMarket, House, Tower, Plaza, Parc,
          Church, Office, Factory, Warehouse, etc.
          (Les numériques type "01", "02" marchent aussi pour rétro-compat.)
weight  = OPTIONNEL — poids de tirage (défaut 10).
          Plus élevé = variante plus fréquente.
ext     = fbx ou glb

Exemples :

FichierDimsPoidsNote
block_1x1_House.fbx25×25 m10 (défaut)Petite maison
block_2x2_Stores.fbx50×50 m10 (défaut)Rangée de commerces
block_2x2_SuperMarket_w30.fbx50×50 m30Supermarché — 3× plus fréquent
block_3x3_Office_w3.fbx75×75 m3Bureaux rare (0.3× le défaut)

Le nom sémantique est le nom qui apparaît dans le générateur ci-dessous, ce qui rend le tuning beaucoup plus intuitif que _01, _02, etc.

Emplacement

fbx/SET/block_<W>x<H>_<NN>[_w<weight>].fbx
glb/SET/block_<W>x<H>_<NN>[_w<weight>].glb

Deux niveaux de poids

Les poids se combinent de façon hiérarchique :

  1. Niveau type (S / M / L / XL / 1×2 / 2×3 / 2×4) — contrôlé par les sliders du générateur ci-dessous. "Je veux 30 % de M dans la ville" → slider M à 30.
  2. Niveau variante — le _w<weight> dans le filename. Quand l'algo choisit "un M", il tire parmi block_2x2_01.fbx, _02_w30.fbx, _03_w3.fbx en proportion des poids individuels.

L'algorithme parse le filename :

const m = filename.match(/^block_(\d+)x(\d+)_(\d+)(?:_w(\d+))?\.(fbx|glb)$/i);
if (m) {
    width   = +m[1];
    height  = +m[2];
    variant = +m[3];
    weight  = m[4] ? +m[4] : 10;
}

Le script build_manifest.py fait ce parsing automatiquement et remplit une section cityBlocks dans le manifest :

"cityBlocks": [
    { "file": "SET/block_1x1_01.fbx",    "w": 1, "h": 1, "variant": 1, "weight": 10 },
    { "file": "SET/block_2x2_01.fbx",    "w": 2, "h": 2, "variant": 1, "weight": 10 },
    { "file": "SET/block_2x2_02_w30.fbx","w": 2, "h": 2, "variant": 2, "weight": 30 },
    { "file": "SET/block_2x3_01.fbx",    "w": 2, "h": 3, "variant": 1, "weight": 10 }
]

Relance python scripts/build_manifest.py quand tu ajoutes / renommes des blocs.

5. 🛠️ Règles de modeling

  • Unités : le fichier est exporté à l'échelle mètre (1 unité Blender = 1 m). Le script export_traits.py gère déjà le scale global.
  • Pivot = coin inférieur-gauche du bloc (X=0, Y=0, Z=0 au coin sud-ouest). Pas centré, pas aligné au bâtiment — vraiment le coin extérieur.
  • Axe Y = haut. Le bâtiment monte vers +Y.
  • Demi-route de 5 m sur chacun des 4 côtés, collée au bord du bloc.
  • Coins : inclure le quart de carrefour (revêtement + coin de trottoir arrondi). Pas de feu de circulation ni de passage piéton — l'algo les ajoutera en instances.
  • Bord extérieur : le trottoir s'arrête pile sur la ligne du bloc. Si un morceau dépasse de 1 cm, il y aura z-fight avec le voisin.
  • Pas de flèches peintes sur la route — le sens est géré en gameplay.
  • Export : GLB via le script existant. Nommer le fichier selon la convention.

6. 🎲 Algorithme de placement

Le placement est glouton pondéré sur grille : on remplit cellule par cellule, en tirant un bloc aléatoire parmi ceux qui rentrent. Garantit zéro trou car le 1×1 rentre toujours.

GRID = 20×20 cellules  = 500×500 m

1. Marquer toutes les cellules comme vides.
2. Générer la liste des cellules vides, dans un ordre aléatoire.
3. POUR chaque cellule (x, y) vide :
     Tirer un bloc pondéré selon [XL:5%, L:10%, 2×4:15%, 2×3:20%,
                                   M:30%, 1×2:10%, S:10%]
     SI rectangle, 50/50 pour tourner à 90°.
     SI le footprint tient (bornes + cellules libres) :
         PLACER le bloc, marquer les cellules occupées.
     SINON :
         Essayer la taille juste en dessous.
         Fallback S (1×1) : rentre toujours si la cellule est vide.
4. Toutes les cellules sont remplies → 0 trou garanti.

Variantes optionnelles pour donner du caractère :

  • Zones thématiques : pondération change avec la distance au centre (centre = L/XL, périphérie = S/M).
  • Avenues forcées : aligner des rangées complètes de M pour créer des rues rectilignes.
  • Spots landmark : forcer un XL à position fixe (mairie, parc).
  • Seed persistante : même seed = même ville (utile pour debug + multiplayer futur).

7. 🗺️ Exemples de layouts possibles

Trois stratégies de placement sur la même grille 10×10, pour illustrer le type de ville qu'on peut obtenir en changeant juste les poids.

Aléatoire pondéré — mix organique de tous les blocs. Mood Crazy Taxi / GTA VC.
Downtown centré — poids biaisé par distance au centre : XL au milieu, S en périphérie. Skyline classique.
Grid + avenue — avenue forcée en croix (pas de bâtiments) + quartiers de M alignés. Feel Manhattan / Mirror's Edge.

9. 🔗 Pipeline complet

1. Modeling (Blender)
   - Modéliser le bloc en respectant les règles §5.
   - Export GLB → glb/CITY/block_WxH_NN.glb

2. Manifest
   - python scripts/build_manifest.py
     → détecte automatiquement les blocs dans glb/CITY/
     → ajoute une section cityBlocks au manifest

3. Runtime (JS)
   - CityGenerator lit le manifest, groupe par (W, H).
   - Génère la grille 20×20, fait le glouton pondéré.
   - Instancie les blocs aux bonnes positions mondiales
     (gridX × 25 m, 0, gridZ × 25 m).

Quand le premier bloc block_2x2_01.glb sera prêt, je câble le CityGenerator et l'intègre à GameWorld.
