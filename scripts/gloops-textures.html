🖼️ Textures & Nomenclature

Convention de nommage des fichiers textures — le nom determine le slot three.js auto-connecte.

1. 📋 Suffixes → Slots three.js

Les textures vivent dans images/chr/<material>/. Le suffixe du fichier determine automatiquement le slot three.js dans lequel la texture est branchee.

SuffixeSlot three.jsUsage
_ID.pngrgbMaskMask RGB — 3 zones de couleur (R, G, B)
_Normal.pngnormalMapNormal map
_Roughness.pngroughnessMapRoughness
_Occ.pngaoMapAmbient Occlusion
_DS.pngdisplacementMapDisplacement Scale
_DW.pngdiffuseWeightMapDiffuse Weight (body uniquement)
_Ramp.pngblendMaskGradient A ↔ B blend
_SSS.pngsssMapSubsurface Scattering
_Bump.pngbumpMapBump (eyes)
_Metalness.pngmetalnessMapMetalness (eyes)
_Emit.pngemissiveMapEmissive
_Alpha.pngalphaMapTransparence
_Pattern.N.pngpatternPattern overlay (variante N, 0..18)
_Diffuse.N.pngdiffuseIris variant (eyes, 0..11)

2. 📁 Structure par materiau

images/chr/<material>/
├── <material>_ID.png          ← RGB Mask
├── <material>_Normal.png
├── <material>_Roughness.png
├── <material>_Occ.png
├── <material>_DS.png
├── <material>_Ramp.png
├── <material>_SSS.png
├── diffuse/                    ← Variantes diffuse (optionnel)
│   └── <material>_Diffuse.N.png
└── pattern/                    ← Patterns overlay (optionnel)
    └── <material>_Pattern.N.png

Remplacer <material> par body, horns, eyes, eyeGlass, etc.

4. 🔌 Auto-connect au chargement

Au boot, autoConnectTextures() lit la section autoConnect de manifest.json et branche chaque texture detectee dans le bon slot du materiau correspondant.

"autoConnect": {
    "body": {
        "rgbMask":       [{ "id": "body/body_ID",   "path": "..." }],
        "normalMap":     [{ "id": "body/body_Normal", "path": "..." }],
        "pattern":       [
            { "id": "body/body_Pattern.0",  "path": "...", "variant": 0 },
            { "id": "body/body_Pattern.1",  "path": "...", "variant": 1 },
            ...
        ]
    }
}

Le systeme est genere par build_manifest.py (a lancer apres chaque ajout de texture).

5. ✅ Ajouter un nouveau materiau

  1. Creer le dossier images/chr/<matName>/
  2. Nommer les textures avec la convention <matName>_Suffixe.png (voir table section 1)
  3. Pour les variantes : <matName>_Pattern.0.png, _Pattern.1.png...
  4. Relancer build_manifest.py
  5. Refresh le navigateur → les textures apparaissent dans le panneau MAT
