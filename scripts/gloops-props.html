🔧 Paired Props

Systeme de props auto-attaches par convention de nommage — lunettes, chapeaux, accessoires.

⚡ Concept en 30 secondes

Quand tu selectionnes une animation (ex: Horns/Hat_CowBoy), le systeme cherche automatiquement un FBX de meme nom dans Horns/PROPS/Hat_CowBoy.fbx. S'il existe, le mesh 3D du chapeau s'attache au personnage.

1. 📁 Structure de dossiers

fbx/ANIM/
├── Horns/
│   ├── Hat_CowBoy.fbx          ← animation (blendshape + bones)
│   └── PROPS/
│       └── Hat_CowBoy.fbx      ← mesh du chapeau (meme nom)
├── Eyes/
│   ├── Aviator.fbx             ← animation yeux
│   └── PROPS/
│       └── Aviator.fbx         ← mesh lunettes

2. 🎯 Les 3 types de props (auto-detectes)

TypeDetectionAttacheCas d'usage
A Static Aucun SkinnedMesh Parente a spine_05 Objet simple (pomme, roche...)
B Mini-rig SkinnedMesh, bones != Gloops (<50% match) Parente a spine_05 Prop avec son propre skeleton
C Rebind SkinnedMesh, bones = Gloops (>50% match) Rebind sur skeleton character Lunettes, chapeaux (full skeleton export) ⭐

Pourquoi Type C est le meilleur

Type C = full skeleton + mesh. Le mesh est lie aux bones du character, donc il suit toutes les deformations automatiquement (head rotation, blendshapes, etc.) sans avoir a choisir un bone parent.

Le rebind utilise les _baseBoneInverses du character (T-pose capturee au load), pas ceux du prop. Donc meme si le prop FBX a ete exporte avec une pose differente, le rebind est correct.

3. 🎨 Export depuis Maya (Batch Prop Export)

  1. Ouvre la scene avec le Gloops skeleton
  2. Selectionne les meshes des props dans le viewport
  3. Dans export_traits.py, set :
    • Skeleton Root = racine du skeleton
    • Target = fbx/ANIM/<Category>/PROPS/
  4. Clique BATCH EXPORT PROPS (bouton bleu)
  5. → Un FBX par mesh, nomme d'apres le mesh, contenant skeleton + mesh, sans animation

⚠️ Le nom du mesh dans Maya doit correspondre exactement au nom du FBX d'animation.

4. ⚙️ Offsets (rotation / position / scale)

Fichier config/paired-offsets.json — applique automatiquement a l'attach :

{
    "Eyes/*":                { "rotation": [0, 0, -90] },
    "Eyes/Pirate.fbx":       { "rotation": [0, 0, -90], "position": [0, 0.05, 0] },
    "Horns/*":               { "rotation": [0, 0, 0] }
}
  • Wildcard Category/* = s'applique a toute la categorie
  • Exact Category/file.fbx = override pour un prop specifique (prioritaire)
  • Rotations en degres (XYZ Euler), positions en unites monde, scale = nombre ou [x,y,z]
  • Type A/B : offset sur le transform du node. Type C : bake dans la geometry (parce que le transform du mesh n'affecte pas les vertices skinnes)

Sliders live dans l'UI

Dans le panneau Props → Paired Prop Offsets, tu as des sliders pour Rotation X/Y/Z, Position X/Y/Z, Scale. Ils modifient l'offset en temps reel.

Bouton 📋 Copy JSON → copie la ligne prete a coller dans config/paired-offsets.json.

5. 🎨 Materiaux partages par categorie

Tous les props d'une categorie partagent les memes materiaux, identifies par leur nom :

  • 13 lunettes Eyes → 2 entries dans MAT : 🔧 Eyes / Glass et 🔧 Eyes / GlassStructure
  • Tu edites un shader une fois → toutes les variantes de lunettes sont mises a jour
  • Multi-material par face supporte (geometry.groups → un slot par materiau)

6. 📜 Script de scan

Apres chaque ajout/suppression de FBX dans un dossier PROPS/, relancer :

python scripts/build_props_manifest.py

→ Met a jour la section pairedProps dans fbx/manifest.json.

⚠️ Attention sync Google Drive

Le manifest.json peut etre ecrase par la sync Google Drive si une ancienne version est remontee. Dans ce cas, relance le script et c'est repare.

7. 👥 NPCs et props

Les NPCs sont clones du player via SkeletonUtils.clone, qui inclut tous les enfants du modele. Pour eviter que les NPCs heritent de tes lunettes :

  • A l'attach, chaque prop est tagge avec userData.isPairedProp = true
  • Apres le clone, NpcGloop._stripPairedProps() walk le clone et supprime tout node tagge
  • → Les NPCs spawnent toujours sans props
