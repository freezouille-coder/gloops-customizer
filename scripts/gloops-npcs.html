👥 NPCs Spéciaux

Gardes fixes, PNJ a missions (Papy Mush), marchands — tous les NPCs qui ne sont pas des wanderers generiques.

1. 🗂️ Les 3 categories de NPCs

CategorieComportementConfig
Wanderer (generique) Spawn random dans une zone, wander, fuit le joueur en voiture, petits dialogues, ecrasable Automatique (spawn aleatoire via zone_spawn_*)
Guard (garde) Fixe sur spawn, attaque le joueur si trop proche, poursuit un certain temps, retourne au spawn config/world-entities.json → guards
Story NPC (PNJ a missions) Fixe, donne des quetes, anim idle, bubble ! quand il a une quete config/world-entities.json → story_npcs

2. 🛡️ Gardes fixes

Concept : un garde a un spawn fixe et attaque si le joueur s'approche trop. Utile pour les batiments sensibles (banque, casino, bordel, labo).

Nommage Maya

Utilise un locator nomme selon ce pattern :

spawn_guard_<type>_<lieu>

Exemples:
  spawn_guard_security_bank
  spawn_guard_bouncer_club
  spawn_guard_cop_plaza
  spawn_guard_doorman_casino
  spawn_guard_labcoat_laboratory

Config JSON

// config/world-entities.json
{
    "guards": {
        "spawn_guard_security_bank": {
            "breed": "zogzog",
            "attack_radius": 5,
            "chase_range": 12,
            "leash_range": 20,
            "faction": "sugarheads",
            "weapon": "club",
            "health": 3,
            "hat": "militar"
        },
        "spawn_guard_bouncer_club": {
            "breed": "zogzog",
            "attack_radius": 3,
            "chase_range": 8,
            "leash_range": 12,
            "hat": "Hat_SmallDeamon"
        },
        "spawn_guard_cop_plaza": {
            "breed": "salaryman",
            "attack_radius": 4,
            "chase_range": 15,
            "leash_range": 25,
            "hat": "Hat_Worker",
            "chase_in_vehicle": true
        }
    }
}

Champs du garde

ChampDefautRole
breed"zogzog"Breed du gloop (voir liste des breeds)
attack_radius5Rayon en metres ou le garde se declenche
chase_range12Distance max de poursuite
leash_range20Au-dela, le garde abandonne et retourne au spawn
faction"sugarheads"Faction du garde (influence couleurs, ennemis)
hatnullHorn animation a appliquer (Hat_Worker, militar...)
health3Nombre de hits avant d'etre KO
weaponnullProp arme si besoin (ex: "club")
chase_in_vehiclefalseSi true, le garde spawn dans une voiture et poursuit

States du garde (FSM)

IDLE          → reste au spawn, scan circulaire, petit swaggering
   |
   | player in attack_radius
   v
ALERT         → bubble ❗, fais face au joueur, 1s de warning
   |
   | still close after 1s
   v
CHASE         → court apres le joueur (anim run_panic ou run)
   |
   | (player within 1.5m)            (player > leash_range)
   v                                  v
ATTACK                              RETURN → retourne au spawn → IDLE
   |
   | (attack animation played)
   v
CHASE (loop)

3. 🧙 Story NPCs (PNJ missions)

Concept : un personnage nomme, fixe, qui donne des quetes au joueur. Papy Mush, Le Maire, Miss Aveggies, etc.

Nommage Maya

spawn_story_<nom>_<lieu>

Exemples:
  spawn_story_papymush_garden
  spawn_story_mayor_cityhall
  spawn_story_miss_aveggies_farm
  spawn_story_dealer_alley
  spawn_story_scientist_laboratory

Config JSON

// config/world-entities.json
{
    "story_npcs": {
        "spawn_story_papymush_garden": {
            "breed": "mush",
            "name": "Papy Mush",
            "dialogue_key": "papymush_intro",
            "missions": [
                "deliver_carrot",
                "collect_10_mushrooms",
                "visit_shaman_temple"
            ],
            "hat": "magician",
            "scale": 1.15,
            "idle_anim": "01_peace.fbx",
            "bubble_emoji": "🍄"
        },
        "spawn_story_mayor_cityhall": {
            "breed": "corpo_suit",
            "name": "Mayor Corpo",
            "dialogue_key": "mayor_intro",
            "missions": ["pay_taxes", "run_errand_for_industry"],
            "hat": "Hat_Ceasar"
        }
    }
}

Champs du story NPC

ChampRole
breedBreed du gloop
nameNom affiche au-dessus de sa tete
dialogue_keyCle dans config/dialogues.json pour le dialogue d'intro
missionsListe des IDs de missions qu'il peut donner (dans l'ordre)
hatAnimation Horns appliquee au spawn
scaleTaille du gloop (1.0 = normal)
idle_animAnimation emotion jouee en boucle
bubble_emojiEmoji qui apparait au-dessus quand il a une quete disponible

Interaction

  1. Le joueur s'approche (dans rayon ~3m)
  2. Bubble ! (quete dispo) ou ? (continue de parler) au-dessus
  3. Touche F → lance le dialogue (lecture depuis dialogues.json)
  4. Fin de dialogue → si mission disponible, elle est acceptee et un marker s'affiche sur la mini-map

Persistance

L'etat du story NPC (missions done, dialogue counter, etc.) est save dans localStorage via :

{
    "story_state": {
        "papymush_garden": {
            "current_mission": "collect_10_mushrooms",
            "missions_done": ["deliver_carrot"],
            "dialogue_counter": 3
        }
    }
}

4. 🎯 Missions (data-driven)

Les missions sont definies dans config/missions.json (fichier a creer) :

{
    "deliver_carrot": {
        "title": "Une carotte pour Mush Junior",
        "description": "Mon petit-fils veut une carotte de Miss Aveggies. Tu peux aller lui chercher?",
        "giver": "papymush_garden",
        "objectives": [
            { "type": "interact", "target": "miss_aveggies_farm" },
            { "type": "return_to", "target": "papymush_garden" }
        ],
        "reward": { "gc": 50, "xp": 10 }
    },
    "collect_10_mushrooms": {
        "title": "Les champignons de la foret",
        "description": "Cueille 10 champignons dans la foret sacree.",
        "giver": "papymush_garden",
        "objectives": [
            { "type": "collect", "item": "mushroom", "count": 10 },
            { "type": "return_to", "target": "papymush_garden" }
        ],
        "reward": { "gc": 100, "item": "shaman_recipe_01" }
    }
}

Types d'objectifs

TypeParamsCompletion
interacttargetLe joueur presse F sur ce NPC/batiment
collectitem, countLe joueur ramasse N items de ce type
killtarget, countLe joueur tue N cibles (gardes, demons)
crushcountLe joueur ecrase N NPCs en voiture (voie corrupted)
deliveritem, targetAller porter un item a un NPC
return_totargetRetourner voir le NPC donneur
reachzoneEntrer dans une zone precise
waitsecondsTimer d'attente (livraison dans 60s)

5. 🗺️ Zone de spawn aleatoire (wanderers)

Les NPCs generiques (non-fixes) spawnent dans des zones, pas sur des locators. Le spawner tire N NPCs aleatoirement dans chaque zone au boot.

Nommage Maya

zone_spawn_<faction>_<lieu>

Exemples:
  zone_spawn_aveggies_forest        ← forest biome → Bobo, Mush, NionNion
  zone_spawn_aveggies_farm
  zone_spawn_sugarheads_plaza       ← urban → SugarHead, CryptoBro, ZogZog
  zone_spawn_sugarheads_industry
  zone_spawn_neutral_station        ← Salary Man
  zone_spawn_rebels_underground     ← Street Pirate
  zone_spawn_mixed_downtown         ← toutes factions melangees

Config (optionnel)

// config/world-entities.json
{
    "spawn_zones": {
        "zone_spawn_aveggies_forest": {
            "count": 8,
            "breed_weights": {
                "bobo": 0.50,
                "mush": 0.35,
                "nionnion": 0.15
            }
        },
        "zone_spawn_sugarheads_plaza": {
            "count": 12,
            "breed_weights": {
                "sugarhead": 0.40,
                "cryptobro": 0.25,
                "zogzog": 0.15,
                "salaryman": 0.20
            }
        }
    }
}

Si un zone_spawn_* n'a pas de config, le code utilise les defauts de la faction (gloops-classes.json).

6. ✅ Checklist workflow

Ajouter un garde

  1. Dans Maya, creer un locator spawn_guard_<type>_<lieu> a la position voulue
  2. Le poser au sol (Y = hauteur terrain) dans le group ZONES ou NPC_SPAWNS
  3. Ajouter sa config dans config/world-entities.json → guards
  4. Exporter le FBX mis a jour → fbx/WORLD/zones.fbx
  5. Refresh → le garde apparait et se comporte selon sa config

Ajouter Papy Mush (ou autre story NPC)

  1. Dans Maya, locator spawn_story_<nom>_<lieu>
  2. Config dans world-entities.json → story_npcs
  3. Ecrire le dialogue dans config/dialogues.json sous la cle dialogue_key
  4. (Optionnel) Ecrire les missions dans config/missions.json
  5. Exporter FBX + refresh

Ajouter un carrefour

  1. Dans Maya, plane zone_intersection_01 couvrant l'intersection
  2. 4 locators trafficlight_north_01, _south_01, _east_01, _west_01
  3. (Optionnel) Config du cycle dans world-entities.json → intersections
  4. Exporter FBX + refresh → feux visibles + voitures qui freinent au rouge
